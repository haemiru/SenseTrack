🔄 한 세션이 흘러가는 방식

카메라 켜기 → 자극 주기 → 반응 측정 → 리포트 → AI 분석 → 저장

1
측정 시작카메라(영상)+마이크(오디오) 권한 요청, 세션 타이머 시작, 매 프레임 분석 루프 가동
2
자극 적용향(라벤더·오렌지…) 또는 소리(싱잉볼 A~G) 선택 후 버튼 클릭 → 반응 대기 모드 진입
3
반응 감지적용 직후 0.5초를 기준점(baseline)으로 잡고, 이후 고개·입·눈썹·미소 변화가 임계값을 넘으면 그 순간을 "반응 시간"으로 기록 (또는 "반응 없음" 수동 처리)
4
리포트 생성호흡 안정도 추이 그래프(Chart.js), 자극별 반응률 카드, 타임라인 자동 생성
5
AI 전문가 의견Gemini가 세션 데이터를 읽고 따뜻한 톤의 한국어 인사이트 3~4문장 작성 (비동기, UI 안 막힘)
6
저장 & 상담 연결리포트를 Firebase에 저장, 카카오 오픈채팅으로 전문가 1:1 상담 연결

📊 측정하는 4가지 지표

전용 의료 센서가 아니라, 카메라·마이크 신호에서 추정하는 휴리스틱 값입니다.

🫁 호흡 안정도

입 움직임(jawOpen)과 눈 깜빡임 패턴으로 추정한 0~100점. 입을 적게 움직이고 깜빡임이 일정할수록 높음.

⏱️ 반응 시간

자극 제시 후 얼굴/표정이 유의미하게 바뀌기까지 걸린 초. 평균 대비 빠름/느림도 표시.

🔄 고개 움직임

랜드마크 거리 비율로 Yaw/Pitch 각도 근사 → 정면/좌·우/상·하 방향 + 각도. EMA로 떨림 보정.

🎤 발성 지속 시간

마이크 RMS가 임계값(0.05)을 넘는 동안을 "발성"으로 보고 지속/최대 시간 측정.

🩺 코드 상태 진단

잘 만든 부분과, 손보면 좋은 부분.

잘 된 점
  • 모듈 책임 분리가 깔끔 — 비전/오디오/세션/AI/DB가 각자 파일, 콜백으로 느슨 결합
  • MediaPipe GPU 실패 시 CPU 폴백 등 방어 로직
  • 노이즈/떨림에 EMA 스무딩 적용 (오디오·고개)
  • 비밀키를 .env(gitignore) + import.meta.env로 안전 처리
  • AI 호출이 비동기라 리포트 표시가 막히지 않음, 실패 시 정적 문구로 폴백
  • 반응 감지를 절대값이 아닌 baseline 대비 변화량으로 판정 (오작동↓)
개선하면 좋은 점
  • 자극 한글 이름 매핑(titleMap/nameMap)이 4곳에 중복 → 상수 1곳으로
  • 미사용 zustand 의존성 제거
  • endSession()에서 faceTracker.destroy() 미호출 (오디오만 정리)
  • .env.example에 VITE_GEMINI_API_KEY 누락 — 추가 필요
  • 테스트·린터·포매터 미설정
  • "호흡/반응"은 추정 휴리스틱 — 의료적 해석 주의 (앱 내 면책 문구는 있음 ✅)
